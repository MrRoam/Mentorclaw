# mentorclaw 浏览器 UI 第一版设计说明

这份文档的目标不是做一个“好看的网站”，而是做一个能让别人一眼看出 mentorclaw 和普通聊天 Agent 不一样的 demo。

要展示的核心不是“它会聊天”，而是：

- 它知道你在学什么
- 它能把目标变成计划
- 它能把计划变成今天的动作
- 它知道你卡在哪里
- 它能把资料、任务、进度放在一个界面里

所以浏览器 UI 第一版必须突出：

1. 聊天不是唯一入口
2. 计划是可见的
3. 任务是可见的
4. 资源是可见的
5. 进度和复盘是可见的

## 产品定位

第一版浏览器 UI 不是替代飞书。

第一版浏览器 UI 的作用是：

- 演示 mentorclaw 的独特价值
- 承载飞书做不好的功能
- 让测试者看到“计划和资源是怎么被组织起来的”

## 第一版必须有的页面

只做 4 个页面就够了。

### 1. 首页 / Dashboard

这是最重要的一页。打开之后，用户要立刻知道：

- 我当前在学什么
- 今天最重要的任务是什么
- 我离目标还有多远
- 我可以马上做什么

页面建议分成 5 个区域：

#### 区域 A：顶部目标卡

显示：

- 当前学习目标
- 截止时间
- 当前阶段
- 总体进度

例子：

- 目标：两周内完成“教育心理学第一章”
- 截止：4 月 16 日
- 当前阶段：计划执行中
- 进度：28%

#### 区域 B：今日任务

显示 3 到 5 个最重要任务。

每个任务要能看到：

- 标题
- 状态
- 截止时间
- 关联资源

例子：

- 看教材第 12-18 页
- 做第 18 页第 1-4 题
- 看网课第 3 节 12:00-18:00

#### 区域 C：风险提醒

显示系统主动发现的问题：

- 已逾期任务
- 连续卡住
- 长时间没有进展
- 某知识点掌握脆弱

这块很重要，因为它能体现 mentorclaw 的“主动性”。

#### 区域 D：最近进展

显示最近发生的事情：

- 新建了计划
- 完成了某任务
- 做了一次测验
- 某个点需要复习

#### 区域 E：继续学习入口

这里放一个醒目的输入框或按钮：

- 继续和 mentorclaw 对话
- 开始今天任务
- 上传新资料

## 2. 聊天页

聊天页不能只是普通对话框，否则别人会觉得这只是另一个聊天机器人。

聊天页建议是“两栏布局”。

### 左边：对话区

正常聊天。

### 右边：当前学习上下文

实时显示：

- 当前 workflow：planning / tutoring / evaluation / review / replanning
- 当前计划标题
- 当前 thread 标题
- 当前任务
- 当前关联资源

这块非常重要，因为它能把“mentorclaw 的脑子”展示出来。

如果只有聊天，不展示这些上下文，别人看不出它的特别之处。

## 3. 计划页

计划页是 demo 的关键页面之一。

别人要一眼看到：mentorclaw 不是在空谈，它真的把目标拆成了结构化计划。

建议分成 4 个部分：

### 区域 A：计划总览

显示：

- 计划标题
- 时间范围
- 当前阶段
- 完成度

### 区域 B：里程碑

用时间线展示：

- 第 1 阶段：明确范围
- 第 2 阶段：学习核心内容
- 第 3 阶段：测验与复盘

### 区域 C：任务列表

任务必须支持状态切换：

- todo
- doing
- blocked
- done

### 区域 D：任务详情

点开一个任务时，要能看到：

- 为什么要做它
- 验收标准
- 相关资源
- 建议顺序

## 4. 资源页

这是以后最有潜力的一页。

第一版先不用做知识图谱，但一定要做“资源和任务的对应关系”。

页面应该能显示：

- 已上传哪些 PDF / 文档 / 链接
- 每个资源被拆成了哪些部分
- 哪些任务引用了哪些资源

理想展示方式：

- 左边资源目录
- 中间资源详情
- 右边关联任务

例如：

- 教材 A
  - 第 1 章
  - 第 2 章
  - 第 18 页习题
- 视频课 B
  - 第 3 节
  - 第 4 节

再显示：

- 任务 “做第 18 页第 1-4 题” 关联教材 A 的 `p18`
- 任务 “看第 3 节 12:00-18:00” 关联视频课 B 的 `lesson3#12:00-18:00`

## 第一版不要做什么

为了速度，下面这些先不要做：

- 桌面端
- 手机原生端
- 复杂权限系统
- 复杂多人协作
- 真正完整的知识图谱可视化
- 太多 fancy 动画
- 很重的社交功能

第一版只需要把 mentorclaw 的核心价值展示清楚。

## 第一版最值得做的 6 个功能

### 1. 新建目标

用户输入：

- 想学什么
- 截止时间
- 每天能投入多久

然后系统生成计划。

### 2. 聊天 + 右侧上下文

让用户一边聊天，一边看到当前计划状态。

### 3. 今日任务清单

让用户看到今天该做什么，不只是“聊什么”。

### 4. 任务状态更新

用户可以把任务标记为：

- 已完成
- 卡住了
- 稍后再做

### 5. 上传资料

哪怕第一版只是把文件列出来，也比没有强。

### 6. 计划页

把计划可视化地展示出来。

## Demo 时最容易打动人的交互

如果你要做一个能在分享里吸引人的 demo，我建议你重点展示这条链：

1. 用户输入一个目标
2. mentorclaw 生成计划
3. 页面立刻出现里程碑和今日任务
4. 用户上传一份教材或课程资料
5. 系统把某个任务和资料对应起来
6. 用户标记一个任务“卡住了”
7. 系统发出风险提示或建议重排计划

这条链比单纯聊天更能体现产品差异。

## 页面风格建议

不要做成传统管理后台，也不要做成纯聊天工具。

建议方向：

- 左侧轻导航
- 中间主工作区
- 右侧上下文栏

视觉气质应该更像：

- 学习操作系统

而不是：

- 企业 IM
- 普通 AI 聊天页面
- 任务管理工具的套皮

## 最小导航结构

建议左侧导航只有这几项：

- 首页
- 聊天
- 计划
- 资源
- 复盘

## 第一版需要哪些假数据

前端同学可以先用这些假数据开工：

- 一个 learner
- 一个 active plan
- 4 到 6 个任务
- 2 个 milestone
- 2 个 resources
- 4 条 recent events
- 1 条 risk alert

## 前端实现顺序

建议顺序如下：

1. 先做页面骨架
2. 再接假数据
3. 再把聊天页做出来
4. 再做计划页
5. 再做资源页
6. 最后做首页联动

## 最后的设计原则

这不是“给 Agent 加个壳”。

这是要让人看到：

- mentorclaw 在组织学习
- mentorclaw 在管理计划
- mentorclaw 在跟踪进度
- mentorclaw 在使用资源

只要 UI 能把这四件事展示出来，这个 demo 就已经有吸引力了。
